线段树&树状数组

区别

  • 线段树

    • 区间查询、区间修改(通过lazy)

  • 树状数组

    • 单点修改、区间查询（实质是查询前缀信息，如果操作具有逆元就可以维护区间信息了）
    • 树状数组同样可以实现区间修改，但是此时实际上维护的是差分数组

分类

  • 普通线段树
  • 权值线段树
  • 势能线段树

例题

HH的项链 / 采花

  • 思维+普通线段树

思路

我觉得这类问题（可离线）一个解法为：

​ 首先对查询离线、排序

​ 然后不断遍历原数组的每一个点，期间更新维护的线段树

​ 如果某个查询的右端点等于查询的右端点，则查询

​ 重点就在于线段树中维护的值，表示的是遍历的过程中某个时刻的状态

对于第一题，我们需要查询给定区间内元素的种类数。

如果我们将查询离线 & 对R排序，当我们遍历到某个位置时，如果我们需要考虑某个区间内物品的种类数，我们只需要将各个元素最后一次出现的位置赋为1，查询这个区间的sum即可。

这里有一些贪心的想法在，对于两个相同的元素$(a_x = a_y,\ x<y)$​​，如果我们已经遍历到i了，并且某个查询区间的$R=i$​​，只要该区间的$L \le y$​​，就一定有这个元素，可以不用考虑$L$​与$x$的关系。

第二题其实与第一题类似，我们需要查询给定区间内元素出现次数大于1的种类数。

我们在上一题中线段树维护的是各个元素最后一次出现的位置，那么我们这里只要维护各个元素倒数第二次出现的位置即可。

前缀和&差分

性质

  1. 在模数为 $n$​ ​​的情况下，对一个数组进行 $k$ ​​次求前缀和操作等于求 $k%n$ ​​​次。
  2. 差分操作是前缀和操作的逆运算，所以对一个数组求 $k$​​​​​ 次求差分等价于求 $-k$​​​ 次前缀和操作
  3. 对一段连续的序列加上一个多项式，如果多项式的最高次数为k，则做k+1次差分运算后，前k+1项有值，最后的余项为0。

例题

智乃酱的子集与超集

  • 多维前缀和

题目

给一个数组($n\le20$)，代表每个物品的价值，定义一个集合的价值为这个集合所有物品的异或值，现有 $m$​ 次查询，每次选出一些物品，求这些物品的子集和超集的价值和

思路

是一个多维前缀和的问题，一个二维前缀和中，有两种方法求一个位置的前缀和。

  1. a[i][j]=a[i][j-1]+a[i-1][j]-a[i-1][j-1]（容斥原理）
  2. for(i=1;i<=n;i++) a[i][j] += a[i-1][j]; for(i=1;i<=n;i++) a[i][j] += a[i][j-1];

第一种方法使用了容斥原理，因此不易向高维拓展。

本题可以看做是一个20维的多维前缀和问题，暂时简化本问题，考虑一个 $n=3$​　​的情况，我们规定$ a[1][0][1] $​​代表只选择了1、3号物品的价值，那么这个集合的子集价值和就应该是 $a[0][0][0]+a[1][0][0]+a[0][0][1]+a[1][0][1]$，就等于三维情况下的前缀和。

具体考虑如何实现，如果要单纯的像上面一样n等于多少就开一个多少维度的数组显然是不实际的，那么最后在写代码的时候就要用20个循环来嵌套了。因此可以用一个数的各位取0/1来代表是否选择各物品。

智乃酱的前缀和与差分

  • 多次前缀和

题目

给一个数组，求这个数组做k次前缀和/k次差分后的值

思路

（其实不是很懂，为什么会联系到卷积，就是说我明白这里“恰好”是卷积的形式，但是是为啥呢）

观察上面可以得到，a数组第k次做前缀和等于一个全为1的数组做k-1次前缀和后得到的数组与a数组做卷积，众所周知，卷积的复杂度是$O(nlogn)$。

如何求出全为1的数组的前缀和数组呢？我们规定 dp[i][j]为第i次做前缀和得到的数组的第j项，那么就有dp[i][j] = dp[i-1][j] + dp[i][j]，和下面这个问题的递推公式相同：“在一个棋盘上移动，从(1, 1)出发，每次只能向下或者向右移动，问移动到(i, j)有多少种移动的方法”，这实际上就是一个高中学的组合数问题，答案就是$C_{i+j}^{i}$​​，那么本题的dp也同样等于这个式子。

根据最上面提到的性质，$k$​​ 可以被化简为 $(k % mod + mod) % mod$​​，那么最后卷一卷就出来答案了。

题目

思路

Just another board game

题目

一个n*m的矩形，每个格子有一个自己的值，有一个棋子初始位置(0, 0)，有两个玩家，A只能横着移动棋子，B只能竖着移动棋子，AB轮流进行k次移动，并且轮到自己移动时也可以选择立马结束游戏，现在A希望使得棋子的最终位置的值最大，B希望使得棋子的最终位置的值最小，求最后的值。

思路

只和k的奇偶性相关：

如果k为奇数，那么最后移动的人就是A，A一定会选择当前行中最大的，那么B在这之前就一定会选择这样的一行：这行里面的最大值是所有行里面最小的。

k为偶数类似。

但是需要注意这两点：

k等于1时，只能在第一行中选择最大的

并且A玩家可以直接在最开始的时候就选择结束游戏，所以需要额外考虑起始位置的值

题目

  1. 往最左边插入一个元素

GCD Game

题目

有n个数字，两个玩家，每次每个人可以挑选一个数字并且用他的一个不等于自身的因数替换，问谁必输

思路

SG函数

对于每个数字而言1为最终态(P-position)，所以$SG(1)=0$​​​​，2可以移动到1，那么$SG(2)=mex{SG(son_2)}=mex{SG(1)}=mex{0}=1$​​​​，同理$SG(4)=mex{SG(son_4)}=mex{SG(1),SG(2)}=mex{0, 1}=2$​​​​，观察后发现SG(x)={x的质因数个数}

那么最终的结果就是各个数的质因数个数的异或了。

Link with EQ

题目

有一列长为n的椅子，第一个人随便坐那，之后的每个人都会选这样一个位置使得 他到最近的人的距离最大

思路

定义$dp[len]$表示在一个长度为len的区间（这个区间最左边的左边一格坐了人，最右边的右边一个也坐了人）内选择座位的情况。

转移：$dp[len] = \begin{cases} 0 & len\le2 \ 1+dp[\lfloor len/2 \rfloor]+dp[len-\lfloor len/2 \rfloor-1] & len>2 \end{cases}$

那么第一个人选择坐在i位置后，总人数$ans[i]=\begin{cases}2+dp[n-i-1] & i\lt3 \ 3+dp[i-2]+dp[n-i-1] & 3\le i\le n-2 \2+dp[i-2] & i\gt n-2 \end{cases}$​

最终的答案就是$\sum_{i=1}^n dp[i]$，这样的复杂度是$O(n^2)$。

观察之后发现其实$\sum_{i=1}^n dp[i]$还能进一步化简为$\begin{cases} 2\times{3(\frac n2-2)+4+\sum_{i=1}^{n-2}dp[i]}&n&1=0 \ 2\times{3(\frac n2-2)+4+\sum_{i=1}^{n-2}dp[i]}+3&n&1=1 \end{cases} $

因此只要处理一下dp的前缀和即可，复杂度$O(n)$

Yiwen with Sqc

题目

给定一个字符串，对于每一个区间中的每一种字母，求其出现的数量的平方和

思路

前缀和

处理有点类似于杭电5的Array，只不过这题不用线段树/树状数组维护罢了

枚举26个字母，记$a_i$为该字符在原字符串中的位置，每项的长度$len_i=a_{i+1}-a_i$​。

枚举每一个终点i，
$$
\begin{aligned}
ans_k&= \sum_{i=0}^{n} \sum_{j=0}^{i-1} len_i\times len_j\times(i-j)^2 \
&= \sum_{i=0}^{n} \sum_{j=0}^{i-1} len_i\times len_j\times(i^2-2ij+j^2) \
&= \sum_{i=0}^{n} \sum_{j=0}^{i-1} len_i\times[i^2(len_j)-2i(jlen_j)+(j^2len_j)] \
&= \sum_{i=0}^{n} len_i\times[i^2(\sum_{j=0}^{i-1}len_j)-2i(\sum_{j=0}^{i-1}jlen_j)+(\sum_{j=0}^{i-1}j^2len_j)]
\end{aligned}
$$
因此我们只要维护三个标记$\begin{cases}l_1=\sum len_j \ l_2=\sum j\times len_j \ l_3=\sum j^2\times len_j \end{cases}$​

最终的答案就是$ans=\sum i^2l_1-2il_2+l_3$​

题目

二维坐标系中有一些点，每个点在以0.5单位每秒的速度扩散，求这些点什么时候会相连

思路

最小生成树

把扩散看作是连线，所需时间实际上就是最小生成树上最长边对应时间。

Counting Stars

题目

给一个数组，有这三种操作

  1. 查询区间[l, r]的和
  2. 给[l, r]中的每一个数的最高位左移
  3. 将[l, r]中的每一个数减去其最低位

思路

线段树/树状数组

如果只考虑操作1、2，显然可以用线段树维护，我们只要维护区间最高位和、区间剩余位即可，考虑操作3，每个数最多只有$\log n$​​​​位1，也就是最多对一个数运用30次操作3，我们额外维护一个bool标记all0表示这段区间内的所有数全部都是0，如果我们的操作3遇到了这样的区间，直接返回即可。

GCD Game

题目

有n个数字，两个玩家，每次每个人可以挑选一个数字并且用他的一个不等于自身的因数替换，问谁必输

思路

SG函数

对于每个数字而言1为最终态(P-position)，所以$SG(1)=0$​​​​，2可以移动到1，那么$SG(2)=mex{SG(son_2)}=mex{SG(1)}=mex{0}=1$​​​​，同理$SG(4)=mex{SG(son_4)}=mex{SG(1),SG(2)}=mex{0, 1}=2$​​​​，观察后发现SG(x)={x的质因数个数}

那么最终的结果就是各个数的质因数个数的异或了。

Square Card

题目

给你两个圆和一个长为a的正方形，正方形会不断旋转，这个正方形会等概率的出现在平面内，如果某一时刻落入了圆1内则得到了基础分数，落入圆2则得到了额外分数，求在得到基础分数的的同时获得额外分数的概率

思路

由于正方形会不断旋转，如果要使得正方形落入圆内，实际上正方形的中心也是一个圆，那么只要处理这两个圆即可。

答案即为相交部分除以圆1面积。

计算可以参考牛客2的Girlfriend

但是要注意特判：正方形的边长大于圆2；两个相离；圆1包含圆2；圆2包含圆1；

Singing Superstar

题目

给你一个模式串T和数个长度不超过30的匹配串S，对于每个S，求出最多有多少个不重叠的个数

思路

字符串哈希/字典树Trie

因为每个匹配串的长度都不超过30，所以可以提前预处理出模式串中所有长度不超过30的串的答案，之后直接输出即可

标答说的是用字符串哈希，但是会T，可能是因为赛后测评机很慢吧，或者是这个做法的复杂度还是不是最优的

如果用Trie的话，也很容易理解，直接看代码就好了
