动态规划

动态规划常常适用于有==重叠子问题==和==最优子结构==性质的问题，并且记录所有子问题的结果，因此动态规划方法所耗时间往往远少于朴素解法。

动态规划有自底向上和自顶向下两种解决问题的方式。自顶向下即==记忆化递归==，自底向上就是==递推==。

使用动态规划解决的问题有个明显的特点，一旦一个子问题的求解得到结果，以后的计算过程就不会修改它，这样的特点叫做==无后效性==

思想：==空间换时间==

通常出现形式：求最值/出现次数

因此，理论上来说，在不考虑时间的要求下，所有的dp问题都可以使用搜索暴力穷举出所有的情况求解

以斐波那契数列为例

  1. 递归写法（画递归树）
  2. 记忆化递归写法（备忘录）
  3. 递推写法

动态规划的过程

  1. 定义状态
  2. 推导状态转移方程
  3. 考虑初始化/输出值

最大连续子序列和

  1. 用暴力 时间复杂度 O(n3)

  2. dp

    1. 定义状态：dp[i] -> 以a[i]为结尾的最大子序和

    2. 状态转移：dp[i] = max(dp[i-1] + a[i], a[i]);

      ​ dp[i] = a[i] + max(dp[i-1], 0);

    3. 初始化：dp[0] = a[i];

    4. 输出值：max{dp[0]..dp[n-1]}

    5. 对空间的优化

  3. dp(前缀和)

    1. sum[i] = sum{a[0..n]}

    2. 定义状态：dp[i] = min{sum[0..i]}

    3. 状态转移：dp[i] = min(dp[i-1], sum[i]);

    4. 初始化：dp[0] = a[i];

    5. 输出值：max{sum[i] - dp[i]} (0<=i<n)

对象原型

在js的对象中，具有一个隐藏的属性 [[Prototype]]，其值为null或者指向一个对象

当在对象中读取一个不存在的属性值的时候，js会尝试在对象的原型中获取该值。（类似于其他语言中的继承关系）

我们可以通过 __proto__属性设置或获取一个对象的原型对象，但是其并不与 [[Prototype]]完全相同

F.prototype

当使用new F()生成一个新对象时，若F.prototype 是一个对象，那么由该构造函数生成的对象的[[Prototype]]就会被赋值为F.prototype所指向的对象

F.prototype 的值要么是一个对象，要么就是 null，其他值都不起作用

每个函数都会有一个默认的prototype对象，这个对象只有一个属性，即指向自身的constructor

因此，如果一个构造函数的prototype属性被覆盖了，那么由这个构造函数构造的对象就会不具有constructor属性，所以通常通过Rabbit.prototype.jumps = true来设置prototype属性而不是覆盖
